# CLI Reference

The import path is still `osmosis` for compatibility. Cerebellum is the project
and release name.

## `python -m osmosis.imatrix_stream`

Streaming weight-only imatrix generator. It reads safetensors one tensor at a
time and writes llama.cpp's legacy imatrix binary format.

```bash
python -m osmosis.imatrix_stream \
  --model <hf-model-id-or-local-safetensors-dir> \
  --output cerebellum_imatrix.dat \
  -v
```

Important options:

- `--model`: Hugging Face model ID or local directory containing safetensors.
- `--output`: output imatrix path.
- `-v`, `--verbose`: print tensor-level entries.

## `python -m osmosis.imatrix_gen`

Standard imatrix generator with optional activation calibration.

```bash
python -m osmosis.imatrix_gen \
  --model <local-or-hf-model-dir> \
  --output cerebellum_imatrix.dat \
  --num-samples 8 \
  -v
```

Important options:

- `--no-calibrate`: skip activation calibration and use weight-only
  sensitivity.
- `--num-samples`: number of built-in calibration prompts to use when
  calibration is enabled.

## `python -m osmosis.cerebellum`

Measured-ablation allocator. It reads `ablation_results.json`, classifies
tensors by measured PPL delta, and writes a `llama-quantize` tensor type file.

```bash
python -m osmosis.cerebellum \
  --ablation ablation_results.json \
  --source-gguf source-f16.gguf \
  --imatrix cerebellum_imatrix.dat \
  --budget-gb 12.0 \
  --base-type Q4_K_M \
  --output tensor_types.txt \
  -v
```

Important options:

- `--ablation`: JSON with `baseline_ppl` and per-test `ppl` entries.
- `--plan`: optional ablation plan JSON used for labels and metadata.
- `--source-gguf`: source GGUF used for `llama-quantize --dry-run` size
  estimates.
- `--budget-gb`: target output size in decimal GB.
- `--base-type`: base llama.cpp quant type, default `Q4_K_M`.
- `--imatrix`: imatrix file used during dry-run size estimation.
- `--objective-weights`: named multi-domain profile or explicit
  `domain:weight` pairs.
- `--output`: tensor type file to write.
- `--quantize-bin`: path to `llama-quantize` if it is not discoverable.
- `--analyze-only`: print classifications and heatmap without allocating.

Built-in objective profiles:

- `general`: `wiki:0.40,code:0.20,math:0.20,dialogue:0.20`
- `code`: `wiki:0.05,code:0.55,math:0.30,dialogue:0.10`
- `chat`: `wiki:0.20,code:0.10,math:0.10,dialogue:0.60`
- `balanced`: `wiki:0.25,code:0.25,math:0.25,dialogue:0.25`

## `python -m osmosis.budget`

Proxy allocator. It reads weight-only multi-depth sensitivity data and writes a
budget-fitting tensor type file.

```bash
python -m osmosis.budget \
  --sensitivity sensitivity_multi.json \
  --source-gguf source-f16.gguf \
  --imatrix cerebellum_imatrix.dat \
  --budget-gb 12.0 \
  --output tensor_types.txt \
  -v
```

Important options:

- `--sensitivity`: JSON generated by `osmosis.sensitivity_stream`.
- `--source-gguf`: source GGUF for dry-run size estimates.
- `--budget-gb`: target output size in decimal GB.
- `--output`: tensor type file to write.
- `--imatrix`: imatrix file passed through to dry-run quantization.
- `--quantize-bin`: path to `llama-quantize`.

## Output Format: `tensor_types.txt`

The allocator writes one override per line:

```text
blk.10.ffn_down.weight=q5_K
blk.11.attn_q.weight=q3_K
```

Pass it directly to llama.cpp:

```bash
llama-quantize \
  --imatrix cerebellum_imatrix.dat \
  --tensor-type-file tensor_types.txt \
  source-f16.gguf \
  output.gguf \
  Q4_K_M
```

## Not Current Public API

Per-tensor runtime backend placement, dashboard orchestration, and model-specific
automation scripts are not part of the public engine interface.
